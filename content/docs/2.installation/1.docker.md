# Déploiement Docker

**Difficulté** : Intermédiaire | **Temps** : 30–45 min

## Prérequis

- Docker 24+ et Docker Compose 2.x
- Domaine avec accès DNS
- Ports 80 et 443 disponibles

---

## 1. Cloner et configurer

```bash
git clone https://github.com/RebelliousSmile/suddenly.git
cd suddenly
cp .env.example .env
nano .env
```

Valeurs minimales à renseigner dans `.env` :

```bash
DEBUG=False
SECRET_KEY=votre-cle-secrete-tres-longue-et-aleatoire
ALLOWED_HOSTS=votre-domaine.com
DOMAIN=votre-domaine.com
POSTGRES_PASSWORD=motdepasse-postgres-securise
SECURE_SSL_REDIRECT=True
```

Générer une `SECRET_KEY` :

```bash
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

---

## 2. Premier déploiement

```bash
# Construire les images
docker compose build

# Démarrer la base de données
docker compose up -d db redis

# Appliquer les migrations
docker compose run --rm web python manage.py migrate

# Créer un compte administrateur
docker compose run --rm web python manage.py createsuperuser

# Démarrer tous les services
docker compose up -d
```

## 3. Certificat SSL (Let's Encrypt)

```bash
# Test en staging d'abord
docker compose run --rm certbot certonly \
    --webroot --webroot-path=/var/www/certbot \
    --email votre@email.com --agree-tos --no-eff-email \
    --staging \
    -d votre-domaine.com

# Si OK, obtenir le certificat réel (sans --staging)
docker compose run --rm certbot certonly \
    --webroot --webroot-path=/var/www/certbot \
    --email votre@email.com --agree-tos --no-eff-email \
    -d votre-domaine.com

docker compose restart nginx
```

---

## 4. Services Docker Compose

Le `docker-compose.yml` inclut :

| Service | Image | Rôle |
|---------|-------|------|
| `web` | build local | Django + Gunicorn |
| `db` | postgres:16-alpine | Base de données |
| `redis` | redis:7-alpine | Cache et broker Celery |
| `celery` | build local | Workers asynchrones |
| `celery-beat` | build local | Tâches planifiées |
| `nginx` | nginx:alpine | Reverse proxy + SSL |
| `certbot` | certbot/certbot | Renouvellement SSL auto |

## 5. Vérification

```bash
# Statut des services
docker compose ps

# Logs applicatifs
docker compose logs -f web
docker compose logs -f celery

# Tests de fédération
curl -I https://votre-domaine.com/
curl "https://votre-domaine.com/.well-known/webfinger?resource=acct:admin@votre-domaine.com"
curl https://votre-domaine.com/.well-known/nodeinfo
```

## Commandes utiles

### Gestion quotidienne

```bash
# Redémarrer un service
docker compose restart web

# Shell dans le conteneur web
docker compose exec web bash

# Shell Django
docker compose exec web python manage.py shell
```

### Maintenance

```bash
# Appliquer des migrations
docker compose run --rm web python manage.py migrate

# Sauvegarde de la base de données
docker compose exec db pg_dump -U suddenly suddenly > backup_$(date +%Y%m%d).sql

# Restauration
cat backup.sql | docker compose exec -T db psql -U suddenly suddenly
```

### Mise à jour

```bash
git pull origin main
docker compose build
docker compose run --rm web python manage.py migrate
docker compose up -d
```

## Dépannage

| Problème | Solution |
|----------|----------|
| Conteneur ne démarre pas | `docker compose logs [service]` |
| Erreur de connexion DB | `docker compose exec db pg_isready -U suddenly` |
| Erreur 502 Bad Gateway | Vérifier que `web` tourne : `docker compose ps` |
| Réinitialisation complète | `docker compose down -v && docker compose build --no-cache && docker compose up -d` |

> `docker compose down -v` supprime les volumes — toutes les données seront perdues.
